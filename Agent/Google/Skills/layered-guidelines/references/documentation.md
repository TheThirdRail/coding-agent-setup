# Documentation

Use this lane when behavior, commands, setup, environment variables, install paths, or usage changes need docs or runbooks.

## Guidelines

- Update usage/setup docs when behavior, commands, environment variables, install paths, or setup flow changes.
- Keep implementation notes concise and avoid duplicating global policy.
- Use relative repo links in Markdown docs when practical.
- Document verification commands and expected outcomes when future agents or users need them.
- Do not document secrets or private local values except as placeholders.
